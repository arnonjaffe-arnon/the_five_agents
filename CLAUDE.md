## זרימה: בקשה לתוכן חדש מהאינטרנט

כשמגיעה בקשה שדורשת מציאת חומר גלם מהרשת, ראובן מתזמר:

**א. הפעלת חן**
ראובן מפעיל את חן עם הנושא / מילות המפתח / סוג המאמר הרצוי.
חן בודקת זיכרון, מחפשת, ושומרת קובץ ב-`Content/`.

**ב. חן מחזירה לראובן**
- שם הקובץ ב-`Content/`
- 1-2 משפטים על המקור
- לינק למקור המקורי

**ג. ראובן מחליט על המשך**
- אם הבקשה כללה גם **שכתוב / פרסום** ← ממשיך אוטומטית: מפעיל יעל → (אם יעל ביקשה תמונות) → יובל → משלב ושומר ב-`Output/`
- אם הבקשה הייתה **"רק מצא לי מאמר"** ← עוצר ומחזיר את הממצאים למשתמש

## תהליך שיתוף פעולה יעל + יובל (מאמר עם תמונות)

כאשר מגיעה בקשה ליצירת מאמר עם תמונות, ראובן מתזמר את התהליך הבא:

**א. הפעלת יעל**
ראובן מפעיל את יעל לכתיבת המאמר. יעל מתבקשת לסמן בתוך הטקסט מקומות שדורשים תמונה בפורמט:
```
{{IMAGE_NEEDED: "תיאור מפורט של התמונה, כולל סגנון רצוי ליובל"}}
```

**ב. יעל מחזירה לראובן**
- את קבצי הפלט (MD + HTML) עם ה-placeholders
- סיכום קצר של המאמר
- רשימת כל ה-placeholders שהיא השאירה

**ג. הפעלת יובל**
ראובן מפעיל את יובל עם כל אחד מה-prompts שיעל ביקשה. יובל מייצר את התמונות ומחזיר את הנתיבים שלהן.

**ד. שילוב התמונות**
ראובן מחליף כל `{{IMAGE_NEEDED: "..."}}` בקבצי ה-MD וה-HTML בתמונה המתאימה:
- ב-MD: `![תיאור](yuval/outputs/YYYY-MM-DD-slug.png)`
- ב-HTML: `<img src="yuval/outputs/YYYY-MM-DD-slug.png" alt="תיאור" style="max-width:100%;">`

**ה. שמירת גרסה סופית**
ראובן שומר את הגרסה הסופית המלאה (עם תמונות) בתיקייה `Output/`.
